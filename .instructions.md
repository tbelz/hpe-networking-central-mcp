# Dynamic DDL & Monitoring Architecture

## Overview

The graph schema uses a two-layer approach:

1. **Bootstrap DDL** ([src/hpe_networking_central_mcp/graph/schema.py](src/hpe_networking_central_mcp/graph/schema.py)) — Static node/relationship tables for the core domain model (Org, Site, Device, ConfigProfile, etc.) and knowledge layer (ApiEndpoint, ApiCategory, EntityType, DocSection, Script).

2. **Generated DDL** (`generated_ddl.json` in the knowledge DB archive) — Dynamically inferred from OpenAPI response schemas by the [schema_generator](src/hpe_networking_central_mcp/schema_generator.py). Applied at startup via `GraphManager.initialize(generated_ddl_path=...)`.

## Key Modules

| Module | Purpose |
|--------|---------|
| `schema_generator.py` | Infer LadybugDB DDL from OpenAPI specs (type mapping, PK detection, relationship inference) |
| `graph/manager.py` | Loads bootstrap + generated DDL, provides `get_schema_description()` via live catalog introspection |
| `graph/schema.py` | Bootstrap DDL constants (`NODE_TABLES`, `REL_TABLES`, etc.) and helper functions |
| `entity_mapping/entities.py` | `entities_from_node_tables()` / `build_registry_from_node_tables()` — DDL-derived entity registry |
| `seeds/populate_monitoring.py` | On-demand seed: ports, radios, clients (auto_run=false) |

## Build Pipeline Flow

`scripts/build_knowledge_db.py` runs on GitHub Actions:

1. Scrape OpenAPI specs from Aruba Central docs
2. Apply bootstrap DDL to knowledge DB
3. Index and populate API endpoints
4. **Generate DDL** from response schemas → `generated_ddl.json`
5. **Entity mapping** using DDL-derived registry (falls back to static)
6. Populate seed scripts

## Schema Versioning

- No more numeric `SCHEMA_VERSION` — replaced by a content hash of the generated DDL
- Access via `GraphManager.schema_hash` property
- API spec change → new hash → full DB rebuild from fresh archive

## LadybugDB Quirks

- `CALL show_tables() RETURN * WHERE type = 'NODE'` is **invalid** — WHERE doesn't work with CALL. Filter in Python after `rows_as_dict()`.
- Always use `IF NOT EXISTS` in DDL for idempotent schema application.
- Use `python3.12` for tests (python3/3.10 lacks `real_ladybug`).
